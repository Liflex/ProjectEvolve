## Experiment Report

**Number:** 179
**Title:** Scroll-to-bottom FAB with new message count indicator
**Type:** Improvement
**Goal:** UI/UX улучшение раздела chat
**Target:** chat.js, chat-section.js, main.css
**Complexity Impact:** +0 файлов (модификация существующих)
**Files Modified:** `ui/static/js/modules/chat.js`, `ui/static/templates/chat-section.js`, `ui/static/css/main.css`

### Results

**What was done:**
1. **`_newMsgCount` tracker** — новый счётчик на каждом tab, инкрементируется при каждом новом сообщении (assistant, tool, error) пока пользователь прокручен вверх. User-сообщения не считаются — пользователь сам их отправил.
2. **`_trackNewMsg(tab)` helper** — метод для отслеживания, вызывается после каждого `messages.push()` (6 мест в WebSocket handler + send logic)
3. **FAB upgrade** — вместо статичного "↓ BOTTOM" показывает "↓ N NEW" когда есть непрочитанные сообщения, с cyan-цветом, пульсирующей анимацией и динамическим title
4. **Auto-reset** — счётчик сбрасывается при клике на FAB, при ручном скролле вниз (transition from `scrolledUp=true` to `false`), и при создании/восстановлении/branching tab
5. **CSS анимации** — `fab-appear` (плавное появление при показе) и `fab-pulse` (cyan-тень пульсация для привлечения внимания к новым сообщениям)

**Working:** yes
**Tests:** skipped — фронтенд UI изменение, нет бэкенд-тестов для покрытия

### Decision

**Result:** KEEP
**Reason:** Полезное UX улучшение чата — паттерн как в Discord/Slack/Telegram. Пользователь видит сколько новых сообщений пропустил, пока был прокручен вверх. Минимальное изменение (43 строки), нет новых зависимостей.
