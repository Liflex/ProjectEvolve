## Experiment 57: Fix chat auto-scroll — preserve scroll position during x-html re-render

**Проблема:** При потоковом выводе в чате, `x-html` полностью заменяет innerHTML сообщений на каждый токен (через `chatTick++`). Это может сбрасывать позицию скролла браузера, из-за чего пользователь, читающий старые сообщения, внезапно перемещается вниз.

**Решение:** MutationObserver на контейнере сообщений — сохраняет относительную позицию скролла (расстояние от низа) и восстанавливает её после каждого обновления DOM.

## Experiment Report

**Number:** 57
**Title:** Fix chat/live-log auto-scroll — MutationObserver preserves scroll position during streaming
**Type:** Bug Fix
**Goal:** Если человек скролит диалоговое окно в RUN experiment или CHAT не должно автоматически слайдить вниз при потоковом выводе
**Target:** `ui/static/js/modules/chat.js`, `ui/static/templates/lab-run.js`
**Complexity Impact:** +0/-0 файлов (изменены 2 существующих)
**Files Modified:** `chat.js`, `lab-run.js`

### Results

**What was done:**
1. **Chat scroll preservation** — добавлен `MutationObserver` на контейнер сообщений. При каждом изменении DOM (x-html re-render на каждый токен) сохраняется и восстанавливается относительная позиция скролла (расстояние от низа). Если пользователь скроллит вверх — позиция остаётся стабильной.
2. **`onChatScroll`** — теперь сохраняет `_distFromBottom` (точное расстояние от низа в px) вместо булева `scrolledUp`.
3. **Live log** — порог детекции скролла увеличен с 30px до 100px; добавлена FAB-кнопка "BOTTOM" для быстрого возврата к авто-скроллу.

**Working:** yes
**Tests:** skipped — UI fix, smoke tests passed (Python imports, FastAPI load, JS syntax)

### Decision

**Result:** KEEP
**Reason:** Решает реальную UX-проблему — пользователь может читать старые сообщения во время потокового вывода без "прыжков" скролла. MutationObserver не конфликтует с существующей логикой `smartScroll`.
