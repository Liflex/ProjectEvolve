## Experiment Report

**Number:** 151
**Title:** Chat smart collapsed preview (conclusion-first extraction)
**Type:** Improvement
**Goal:** UI/UX улучшение раздела chat
**Target:** chat.js — collapsed message preview rendering
**Complexity Impact:** +0 файлов (изменения в 2 файлах)
**Files Modified:** `ui/static/js/modules/chat.js`, `ui/static/css/main.css`

### Results

**What was done:**
Ранее при сворачивании (FOLD) ассистент-сообщений показывались первые 300 символов контента — часто это заголовок или приветствие, не дающие понимания сути ответа. Теперь:

1. **`_buildSmartCollapsedPreview(content)`** — анализирует контент и извлекает 3 компонента:
   - **title** — первая строка (заголовок/начало)
   - **summary** — статистика скрытого контента: "2 code blocks · 45 lines · 3 more sections"
   - **conclusion** — последний нетривиальный параграф (заключение/итог ответа)
2. **Обновлён рендеринг свёрнутых сообщений** — показывает title → summary → conclusion вместо простой обрезки. Пользователь видит начало, понимает объём скрытого, и читает вывод.
3. **CSS стили** — `.smart-preview-*` классы с визуальным разделением: title (обычный текст), summary (компактная серая строка-разделитель), conclusion (приглушённый текст).

**Working:** yes
**Tests:** JS logic проверена через node.js — все assertions pass (title extraction, conclusion extraction, summary generation, edge cases: empty, short, code-only)

### Decision

**Result:** KEEP
**Reason:** Улучшение существующей функции (FOLD) без добавления нового UI. Меньше кода, больше пользы при сканировании длинных conversation'ов.
