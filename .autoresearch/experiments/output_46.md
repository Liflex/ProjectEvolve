**Важное замечание:** Alpine.js directives (`x-show`, `x-model`, `@click`, `x-text`, `x-html`, etc.) работают ТОЛЬКО когда они обрабатываются Alpine при инициализации DOM. Если HTML подставляется через `x-html`, Alpine directives внутри **НЕ будут работать**.

**Правильный подход для Alpine.js:** Использовать **`Alpine.component()`** или просто держать HTML в index.html, но разбить JavaScript (app.js) на модули.

Поскольку Alpine.js 3.x не поддерживает динамические компоненты (как Vue/React), единственный способ разбить HTML — это **include-подход на уровне сервера** (SSR) или **JavaScript template loader**, который загружает HTML фрагменты и инициализирует Alpine.

Однако, есть нюанс — Alpine CDN `defer` означает, что Alpine инициализируется после парсинга DOM. Если мы используем `fetch()` для загрузки шаблонов (async), Alpine может инициализироваться ДО завершения загрузки.

**Решение:** Использовать `<script>` без defer/async для загрузчика, который:
1. Блокирует парсинг
2. Синхронно (или через await) загружает шаблоны
3. Вставляет в DOM
4. Только потом Alpine (с defer) инициализирует дерево

Но синхронный `fetch` — deprecated. Лучший подход: **использовать `document.addEventListener('alpine:init', ...)`** — это Alpine hook, который вызывается перед инициализацией директив. В этом хуке можно загрузить шаблоны асинхронно и подождать их.
