# AutoResearch

Автономная система исследования и улучшения проектов с помощью AI.

## Что это такое?

AutoResearch — это универсальный инструмент для запуска AI-агента на любом проекте. Агент autonomously анализирует код, предлагает улучшения, вносит изменения и документирует результаты.

## Возможности

- 🤖 **Автономные эксперименты** — AI самостоятельно исследует и улучшает проект
- 📝 **Накопление контекста** — каждая итерация видит результаты предыдущих
- 🎯 **Универсальность** — работает с любым проектом (Python, JS, Go, etc.)
- ⚙️ **Интерактивная настройка** — простой опросник для первичной конфигурации
- 🔄 **Итеративное развитие** — агент учится и улучшается с каждым экспериментом

## Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/autoresearch.git F:/IdeaProjects/autoresearch
   ```

2. **Установите Claude CLI:**
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

3. **Готово!**

## Использование

### Базовый запуск

```bash
# Запуск в интерактивном режиме (настроит и запустит)
python autoresearch.py

# Запуск на текущем проекте
python autoresearch.py --project .

# Указать количество итераций
python autoresearch.py --project . --iter 10

# Указать интервал между итерациями (в минутах)
python autoresearch.py --project . --iter 5 --timeout 2
```

### Только настройка

```bash
# Первичная настройка проекта
python autoresearch.py --project . --configure

# Перенастройка проекта
python autoresearch.py --project . --reconfigure
```

### Примеры

```bash
# 10 экспериментов над Spec Kit с интервалом 5 минут
python autoresearch.py --project F:/IdeaProjects/spec-kit --iter 10 --timeout 5

# Быстрый тест: 3 эксперимента с интервалом 1 минута
python autoresearch.py --project . --iter 3 --timeout 1

# Долгий сеанс: 50 экспериментов с интервалом 10 минут
python autoresearch.py --project . --iter 50 --timeout 10
```

## Структура в проекте

После запуска AutoResearch создаёт в проекте:

```
your-project/
├── .autoresearch/
│   ├── .autoresearch.json   # Конфигурация проекта
│   ├── experiments/
│   │   ├── prompt_1.md      # Промпт 1-го эксперимента
│   │   ├── output_1.md      # Результат 1-го эксперимента
│   │   ├── accumulation_context.md  # Накопленный контекст
│   │   ├── changes_log.md   # Лог всех изменений
│   │   └── summary.json     # Итоговая сводка
│   └── logs/
│       └── autoresearch.log  # Логи запуска
```

## Файл конфигурации

`.autoresearch.json` создаётся при первой настройке:

```json
{

  "goals": [

  "tech_stack": ["Python", "FastAPI", "PostgreSQL"],

```

## Как это работает

1. **Настройка** — первый запуск предлагает указать цели и ограничения проекта
2. **Анализ** — AI изучает структуру проекта и код
3. **Эксперимент** — агент предлагает изменение, реализует его, тестирует
4. **Отчёт** — результаты сохраняются в структурированном формате
5. **Повтор** — следующая итерация учитывает результаты предыдущей

## Советы

1. **Начните с малого** — попробуйте 3-5 экспериментов для теста
2. **Укажывайте чёткие цели** — чем конкретнее, тем лучше результаты
3. **Проверяйте изменения** — каждый эксперимент создаёт backup branch
4. **Читайте отчёты** — `.autoresearch/experiments/changes_log.md` содержит все изменения
5. **Настраивайте интервал** — для сложных проектов увеличьте время между экспериментами
